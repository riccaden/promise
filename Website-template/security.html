Sicherheit

Deine Geschichte
gehört nur dir.

Was du mit Oblivio teilst, ist das Intimste, das du besitzt. Wir behandeln es entsprechend.

01

Verschlüsselung

Alle Verbindungen sind TLS-verschlüsselt. Daten werden in Supabase mit Encryption at Rest gespeichert. Niemand kann sie im Klartext lesen.

02

Row Level Security

Supabase RLS stellt sicher, dass jede Datenbankabfrage nur deine eigenen Daten zurückgibt. Es ist technisch unmöglich, fremde Daten abzurufen.

03

JWT-Authentifizierung

Der Zugang zu deinem Konto ist durch signierte JSON Web Tokens gesichert. Abgelaufene oder ungültige Token werden automatisch abgelehnt.

04

Schweizer Infrastruktur

Das Frontend wird über einen zertifizierten Schweizer Hosting-Anbieter betrieben. Das Schweizer Datenschutzgesetz (revDSG) gilt als Mindeststandard für alle Datenverarbeitungen.

05

Kein Tracking, keine Werbung

Oblivio verwendet keine Werbe-Tracker, keine Analytics-Dienste von Drittanbietern und verkauft keine Nutzerdaten, an niemanden.

06

Recht auf Löschung

Du kannst jederzeit die Löschung deiner Daten verlangen. Dein Konto, deine Gespräche und dein Vermächtnis werden vollständig und unwiderruflich entfernt.

Authentifizierung & Zugang

Der Zugang zu Oblivio erfolgt ausschliesslich über eine verifizierte E-Mail-Adresse in Kombination mit einem selbst gewählten Passwort. Die Passwörter werden niemals im Klartext gespeichert. Supabase Auth verwendet bcrypt-basiertes Hashing, einen der sichersten Standards für Passwortspeicherung.

Sitzungen werden durch kurzlebige JWT-Tokens verwaltet. Nach einem bestimmten Zeitraum werden diese automatisch erneuert oder invalidiert, sodass abgelaufene Sitzungen keinen Zugang gewähren.

Row Level Security (RLS) ist auf allen Tabellen aktiviert. Das bedeutet: Selbst wenn ein technischer Fehler in der Applikationslogik auftreten sollte, würde die Datenbank auf Ebene des Datenbankservers sicherstellen, dass kein Nutzer auf fremde Daten zugreift.

Datenminimierung

Oblivio erhebt nur, was für den Dienst notwendig ist: Name, E-Mail-Adresse, Sprachpräferenz und der Inhalt deiner Gespräche mit dem Biografen. Es werden keine Standortdaten, Gerätefingerprints, Verhaltensmuster oder sonstige Metadaten erhoben, die über die technische Notwendigkeit hinausgehen.

Gesprächsinhalte werden ausschliesslich für die Generierung deines Vermächtnisses verwendet und keiner anderen Analyse, keinem Training von KI-Modellen durch Dritte und keiner Weitergabe unterzogen.

Umgang mit KI-Anfragen

Gesprächsinhalte werden zur Verarbeitung an die OpenAI GPT API übertragen. OpenAI hat sich vertraglich verpflichtet, API-Anfragen von Business-Kunden nicht für das Training ihrer Modelle zu verwenden. Die Übertragung erfolgt ausschliesslich verschlüsselt (TLS 1.3) und nur für den Zeitraum der Verarbeitung. Es findet keine dauerhafte Speicherung bei OpenAI statt.
